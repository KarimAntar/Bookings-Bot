Management overview

Bookings Bot

A faster, consistent first review of booking screenshots in Slack

Implementation & deployment preparation

01 / The opportunity

The Current Problem

The same first review takes time and can produce different answers.

1

Manual time

Agents stop to inspect every screenshot.

2

Repeated checks

The same visible details are checked again and again.

3

Inconsistent decisions

First-pass outcomes can vary by reviewer.

4

Standard first pass

The bot applies one repeatable review method.

02 / The method

The New Method

A straight, four-step path keeps the work easy to understand.

Step 1Agent posts screenshotInside the normal Slack workflow
Step 2Bot receives itThe request stays in the channel
Step 3AI reviews visible detailsOnly what the image clearly shows
Step 4Bot replies in threadA concise first-pass result

03 / How it connects

Simple Architecture

The bot uses the existing Google VM and a secure outbound connection—no public webhook.

WorkplaceSlackScreenshot and request
ServiceBookings BotRuns on the Google VM
ReviewGeminiChecks visible booking details
ResultSlack replyReturns to the same thread

Secure connection from the VM; no public inbound endpoint is required.

05 / Safe boundaries

Human Control

Four clear conditions always return the decision to a person.

!

Unclear image

Blur, crop, or unreadable text → human review.

!

Missing information

A required visible detail is absent → human review.

!

Low confidence

The result is not clear enough → human review.

!

AI or network failure

The service cannot complete safely → human review.

07 / Protection by default

Security

Private GitHub

✓

VM-only secrets

✓

No screenshots committed

✓

Safe logs
